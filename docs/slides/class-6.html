Deployment Options

Once your model is on the Hub, you have three options:

  1. Direct usage via transformers Users download and run locally (free)
  2. Inference Endpoints Serverless API for production (pay-per-use, scales to zero)
  3. Spaces (Gradio demo) Interactive web UI for demos (free tier available)

HuggingFace Authentication

You need a HuggingFace token with "write" permissions

Required for pushing models and creating deployments

  1. Create account at huggingface.co
  2. Go to Settings ‚Üí Access Tokens
  3. Create token with "write" permissions
  4. Keep it secret!

What is a Model Card?

Your primary contribution to the community

  • Explains *what* your model does, *how* it was built, and *why*
  • Makes your work useful and accessible to others
  • Promotes reproducibility and transparency

Model Card Structure

Essential sections:

  1. Model Description: What it is, and what is its architecture?
  2. Training Details: What data did you use? What were your hyperparameters?
  3. Performance: How well does it work? What are its limitations?
  4. Usage: How can others use it?

Pre-Deployment Checklist

Before pushing to Hub:

  • ‚òë Training completed successfully
  • ‚òë Model evaluation done (WER calculated)
  • ‚òë Tested inference locally
  • ‚òë Model card prepared
  • ‚òë HuggingFace token ready

Post-Deployment Testing

After pushing to Hub:

  1. Load from Hub: Verify model downloads correctly
  2. Test inference: Run transcription on sample audio
  3. Check model card: Renders properly on Hub
  4. Test pipeline: Use transformers pipeline integration

Common Deployment Issues

Model fails to load

  • Check all files uploaded
  • Verify config.json is valid
  • Ensure asr_*.py files present

Inference errors

  • Test locally first
  • Check transformers version
  • Verify audio format support
