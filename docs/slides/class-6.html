What is a Model Card?

Documentation for your model

  • What it does
  • How it was trained
  • Performance metrics
  • Usage examples
  • Limitations and biases

Why important?

Helps others understand and use your model professionally

Model Card Structure

Essential sections:

  1. Model Description: What it does, architecture, key features
  2. Training Details: Dataset, time, hardware, cost
  3. Performance: WER score, test set, baselines
  4. Usage: Code examples, installation, I/O
  5. Limitations: Known issues, ethical considerations

Post-Deployment Testing

After pushing to Hub:

  1. Load from Hub: Verify model downloads correctly
  2. Test inference: Run transcription on sample audio
  3. Check model card: Renders properly on Hub
  4. Test pipeline: Use transformers pipeline integration

Common Deployment Issues

Model fails to load

  • Check all files uploaded
  • Verify config.json is valid
  • Ensure asr_*.py files present

Inference errors

  • Test locally first
  • Check transformers version
  • Verify audio format support
